anyclaude

A local proxy that makes Claude Code work with any LLM provider — no config file changes, no vendor lock-in.

claude → anyclaude proxy OpenAI Gemini DeepSeek Groq Ollama

iWhat it is

anyclaude is a lightweight local proxy that sits between your Claude Code CLI and any OpenAI-compatible LLM provider. It translates Anthropic's API format into the target provider's format, so you can run claude against GPT-4o, Gemini 2.5, DeepSeek V4, Groq, Ollama, OpenRouter, xAI, Mistral, or any custom endpoint — without touching a single Claude Code config file.

The key insight: Claude Code speaks the Anthropic Messages API natively, but it also respects ANTHROPIC_BASE_URL. Point that at the anyclaude proxy and the proxy handles the rest — model routing, authentication, format translation.

BYOK design. Bring your own API key for each provider. No central service, no usage tracking, no middleman — the proxy runs entirely on your machine.

1Quick start

terminal
# Install & run — one command
npx anyclaude

# Or pick a specific model
npx anyclaude --model gpt-4o
npx anyclaude --model gemini-2.5-pro
npx anyclaude --model deepseek-v4-pro

Set ANTHROPIC_BASE_URL=http://localhost:PORT and your provider's API key as the ANTHROPIC_AUTH_TOKEN. Then run claude as normal — it routes through the proxy.

Under the hood, anyclaude uses litellm for provider routing — so any provider litellm supports is fair game.

$Why it exists

Claude Code is the best coding agent CLI out there. But what if you want to use it with a model that isn't Anthropic? The /model picker is hardcoded to Anthropic's tiers, and the ANTHROPIC_* env vars only work with Anthropic-compatible endpoints.

anyclaude fills that gap: it's a translation layer, not a wrapper CLI. You keep the real claude binary, your existing sessions, your Anthropic subscription — and layer other providers on top without conflict.
