kernelpath

An exhaustive, visual, free roadmap from first principles to contributing to the Linux kernel — built for self-taught engineers who want systems-level depth.

iWhat it is

kernelpath is a complete curriculum that covers everything from CPU caches to kernel module development. 262 modules across 30 tracks, split into two parts:

Part I — Kernel Path

12 tracks covering operating systems, the Linux kernel specifically (process scheduler, memory manager, VFS, device drivers, networking stack), and a contribute playbook that walks through your first kernel patch.

Part II — CS Core

18 tracks covering the equivalent of an MIT/Stanford CS degree: algorithms, data structures, compilers, networking, databases, distributed systems, architecture, and more. 2702 concepts, 2399 hand-curated resources.

Every module has a mermaid diagram, interactive SVG visualizations (36 abstract-concept visualizers), and ⌘K search across the entire curriculum. Built for Karthik's younger brother — and open for everyone.

2Standout features

Algorithm Visualizer

30 algorithms with code-beside-viz and line-by-line explanation. Sorting, graph algorithms, dynamic programming — interactive in the /sandbox.

17 sandbox tools

CPU simulator, logic gates, IEEE-754 explorer, cache simulator, scheduler visualizer, paging simulator, DFA explorer, Turing machine, gradient descent, k-means, hash/cipher tools, and the SQL sandbox.

/contribute playbook

A concrete guide from finding your first kernel bug to getting a patch merged into mainline Linux.

Curriculum depth

Resources pulled from MIT OCW, Stanford, and industry — not random blog posts. Every concept is referenced to a source you can trust.
