memsweep

A terminal memory monitor that shows what's eating your RAM — and safely reclaims idle Claude Code jobs, Chrome processes, and anything else that's hogging memory.

iThe problem

Claude Code sessions left idle don't die gracefully. kill -9 doesn't always work because the supervisor respawns them. Chrome tabs accumulate. Swap fills up. The only way to actually close an idle Claude session is to unpin it first — and then terminate it.

memsweep automates this. It's a Textual TUI that shows your RAM, swap, and compressor pressure — groups processes into bundles (Claude, Chrome, VS Code, etc.) — and can auto-reclaim memory from idle Claude jobs by unpinning them first, then terminating. The sequence that actually sticks.

1Install

terminal
pip install memsweep

# Or clone & run
git clone https://github.com/l69d/memsweep
cd memsweep && python3 -m memsweep

2Usage

terminal
memsweep             # Launch the interactive TUI
memsweep --report    # One-shot memory report, no TUI
memsweep --reap      # Find and close idle Claude jobs

★What makes it different

Claude-aware

Knows about Claude Code's pin/supervisor system. Unpins before killing — the sequence that actually closes sessions.

Process groups

Groups by app bundle — Claude, Chrome, VS Code, and others — so you see at a glance who's consuming what.

Compressor-aware

Shows memory-compressor and swap pressure. When macOS starts compressing heavily, it's time to reap.

27 tests

Tested against real macOS memory pressure scenarios. Not a hacky script — a proper tool.

Why built

Because killing a Claude PID doesn't close Claude — the supervisor brings it back. memsweep was built to solve that exact frustration.
